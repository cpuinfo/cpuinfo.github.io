How to use radamsa for mutating inputs?

Purpose

In this article we will expore a popular data mutation tool called radamsa. Data mutation is needed for carrying out fuzzing. Hence radamsa is all called as a general purpose fuzzer.

Overview of radamsa

Every software programs take some data as input. If the software is written with good santization of inputs, it can be protected against various attacks. Examples of sanitization could be done with following checks:

  1. Input data size
  2. Input data type check eg. string, float, integer etc, example a credit card number should only have numbers
  3. Input structure check eg. JPG file format
  4. Input with filters for unwanted characters eg. HTML tags in input fields of Web applications to avoid injection attacks

This is not a exhaustive list but provides a good understanding the purpose of input sanitization. So the question is, how do we generate such variant input data for testing the developed software. That’s where radamsa plays a crucial role. It takes an sample input and mutates it with various mutation strategies. The mutataion strategies supported by radamsa can be viewed using the following command:

1

radamsa -l

Here is the sample output of the command.

ab:enhance silly issues in ASCII string data handling
bd:drop a byte
bf:flip one bit
bi:insert a random byte
br:repeat a byte
bp:permute some bytes
bei:increment a byte by one
bed:decrement a byte by one
ber:swap a byte with a random one
sr:repeat a sequence of bytes
sd:delete a sequence of bytes
ld:delete a line
lds:delete many lines
lr2:duplicate a line
li:copy a line closeby
lr:repeat a line
ls:swap two lines
lp:swap order of lines
lis:insert a line from elsewhere
lrs:replace a line with one from elsewhere
td:delete a node
tr2:duplicate a node
ts1:swap one node with another one
ts2:swap two nodes pairwise
tr:repeat a path of the parse tree
uw:try to make a code point too wide
ui:insert funny unicode
num:try to modify a textual number
xp:try to parse XML and mutate it
ft:jump to a similar position in block
fn:likely clone data between similar positions
fo:fuse previously seen data elsewhere
nop:do nothing (debug/test)

Also note that radamsa is also input generator apart from its capability to mutate inputs.

# Mutate the string 'cpuinfo'
echo "cpuinfo" | radamsa

# Generate 10 mutations for the string 'cpuinfo'
echo "cpuinfo" | radamsa -n 10

# Generate 10 mutations for the string '1+2'
echo '1+2' | radamsa -n 10

# Mutate number '1'
echo 1 | radamsa

# List all the mutation options you can use with radamsa
radamsa -l

# Use random engine to generate inputs
radamsa -g random

# Generate a gzip file and mutate using radamsa 
gzip -c /bin/bash | radamsa -n 1000 | gzip -d > /dev/null

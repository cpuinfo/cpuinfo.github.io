Common file formats for executables/ binaries

NameOperating SystemUsage
Executable and Linkable Format (ELF)LinuxExecutables, Object code (.o), Shared Objects (.so), Kernel Objects (.ko),
Portable Executable (PE)WindowsExecutables (.exe), Object code (.o), DLLs (.dll)

Jargon

Other names: executable code, executable file, executable program, executable, binary

Pre-requisite installations

Mingw-w64

Mingw-w64 is an advancement of the original mingw.org project, created to support the GCC compiler on Windows systems. It also has a port for Linux to generate Windows executables.

1
2

sudo apt-get install mingw-w64-x86-64-dev
sudo apt-get install mingw-w64-i686-dev

Exercise #1: Analyze an intermediate Object File Format

Compile add.c using following command. This will generate the intermediate object file add.o. Use the file command to note the file type.

1
2
3

gcc -c -o add.o add.c

file add.o add.c

As it is observed the add.o is stored in ELF format and add.c is a ASCII format as it holds the source code.

Note: Since add.o is an intermediate object file we will be unable to execute it.

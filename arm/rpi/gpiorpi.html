Overview

Options to access GPIOs

There are several options access the GPIO pins on a Linux board. On the Raspberry Pi we have multiple naming conventions for the GPIOs. The most common way of naming is the broadcom pin numbers called BCM and sometimes you can also use physical pin numbers. Most libraries would allow you to declare the numbering scheme before using them.

Installation of lg lbirary

We are going to use the lg library which allows us to access GPIOs on Single Board Computer (SBC) based on Linux. To install the library you can visit the website which details about the installation process. For simplicity I have listed below the commands that will install the library on the Raspberry PI.

#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>

#include <lgpio.h>

#define LFLAGS 0

#define LED_PIN 23

int main(int argc, char *argv[])
{
    int handler;

    int i;
    handler = lgGpiochipOpen(0);

    if (handler >= 0)
    {
        if (lgGpioClaimOutput(handler, LFLAGS, LED_PIN, 0) == LG_OKAY)
        {
            while (1)
            {
                lgGpioWrite(handler, LED_PIN, 0);
                sleep(1);
                lgGpioWrite(handler, LED_PIN, 1);
                sleep(1);
            }
        }

        lgGpiochipClose(handler);
    }
    else
    {
        printf("Error opening the GPIO");
    }
}

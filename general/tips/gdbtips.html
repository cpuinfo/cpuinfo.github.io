Popular GDB plugins

Descriptions below are shamelessly copied from the home page of each plugin

Pwndbg

pwndbg (/poʊndbæg/) is a GDB plug-in that makes debugging with GDB suck less, with a focus on features needed by low-level software developers, hardware hackers, reverse-engineers and exploit developers.

1
2
3

git clone https://github.com/pwndbg/pwndbg
cd pwndbg
./setup.sh

Peda

PEDA - Python Exploit Development Assistance for GDB

1
2

git clone https://github.com/longld/peda.git ~/peda
echo "source ~/peda/peda.py" >> ~/.gdbinit

GEF

GEF (pronounced ʤɛf - “Jeff”) is a set of commands for x86/64, ARM, MIPS, PowerPC and SPARC to assist exploit developers and reverse-engineers when using old school GDB. software developers, hardware hackers, reverse-engineers and exploit developers.

1

 bash -c "$(curl -fsSL http://gef.blah.cat/sh)"

GEF tips

Window redirect in GEF

Use two terminal windows.

In terminal 2 use following command to find the tty

1

tty

The output in my case is /dev/pts/5

In terminal 1 use following commands:

1
2

gdb-gef mybinary
gef➤  gef config context.redirect /dev/pts/5

Then in terminal 1 use the following command on gef shell

Grow the stack downwards

1

gef config context.grow_stack_down True
